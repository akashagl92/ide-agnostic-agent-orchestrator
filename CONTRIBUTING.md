# Contributing

## Principles
- Keep core IDE-agnostic.
- Do not bypass policy checks in adapters.
- Preserve backward compatibility for wrapper entry points.

## Development
1. Open an issue describing scope and risk.
2. Keep changes modular: core vs adapters vs docs.
3. Add/update tests for schemas/contracts.
4. Update docs when behavior changes.

## Pull Requests
- Include summary, risk notes, and validation output.
- If changing policy/event formats, include migration guidance.
